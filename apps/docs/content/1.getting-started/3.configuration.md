# Configuration

VeloceVue allows you to customize the theme colors and other settings through a configuration file.

## Creating the Configuration File

Create a `ui.config.ts` file in your project root with the following structure:

```typescript
import { config } from "veloce/config";

const themeConfig = {
  colors: {
    primary: "#F5276C",
    secondary: "#F54927",
    accent: "#F5B027",
  },
};

export default config(themeConfig);
```

## Using the Configuration

Import and use the configuration in your `main.ts` file:

```typescript
import { createApp } from "vue";
import App from "./App.vue";
import UIConfig from "./ui.config";

// Import VeloceVue styles
import "veloce/style.css";

const app = createApp(App);

// Apply UI configuration
app.use(UIConfig);

app.mount("#app");
```

## Available Color Options

You can customize the following colors:

- `primary` - Primary brand color
- `secondary` - Secondary brand color
- `accent` - Accent color
- `success` - Success state color
- `info` - Info state color
- `warning` - Warning state color
- `error` - Error state color

## Using Custom Colors in Components

Once configured, you can use the custom colors in your components:

```vue
<template>
  <Button severity="primary" label="Primary Button" />
  <Button severity="secondary" label="Secondary Button" />
</template>
```

## Using CSS Variables

The configured colors are available as CSS variables:

```css
.custom-element {
  background-color: var(--ui-color-primary);
  color: var(--ui-color-secondary);
}
```

Or in Vue templates:

```vue
<template>
  <div :style="{ backgroundColor: 'var(--ui-color-primary)' }">
    Custom styled element
  </div>
</template>
```

## Example Configuration

Here's a complete example with all available color options:

```typescript
import { config } from "veloce/config";

const themeConfig = {
  colors: {
    primary: "#F5276C",
    secondary: "#F54927",
    accent: "#F5B027",
    success: "#10B981",
    info: "#3B82F6",
    warning: "#F59E0B",
    error: "#EF4444",
  },
};

export default config(themeConfig);
```
